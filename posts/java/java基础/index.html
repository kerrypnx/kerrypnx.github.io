java的基本内容

java类的形式

1. 包的就是一个文件夹, 一个包里的意思就是一个路径下的
2. String和String buffer的区别: String是常量，一旦创建了就不能修改了，每次的变更其实是先回收，后创建，有时候JVM也会自动转换成STRINGBUFFER
3. 在使用字符串频繁的变更的时候，尽量用stringbuffer
4. Java中所有对对象的引用都是地址
5. 原生类型是实打实的内存
6. 引用数据类型是地址
7. Java方法重载：多个同名函数
8. Build pattern
9. - 给一个类创建一个classBuilder，然后就不访问class了，通过访问classbuilder间接的访问class
10. 类，一般有私有和公开两种，类的路径就是类的包的名字，包的名字也可以用类的路径代替。
11. 访问修饰符：
    a. default  （只有同包的才可以访问）不是一个包里的就不可以访问，同类，同包可以访问，同包的子类可以访问，不同包的子类不能访问
    b. private 只有同类可以访问
    c. protected  只有不同包的时候才不可以访问，如果不同包，继承了以后，也可以访问
12. 类的覆盖，就是有的父类，自己又需要这个名字又需要做一些特殊的设定，比如一个美国人继承了人的类，但是他有个打招呼的方法，他需要说hello，中国人是说你好，就可以覆盖人的类
13. 工厂方法：
    a. 构造器： new class
    b. 工厂方法：有名字的方法 pulic static className methodName () {}
14. printf: 可以格式化输出内容 ‘32.5f’
15. printin: 会换行，和print一样，直接输出内容
16. 模板方法和接口有什么区别？？？？
17. 模板方法就是类的方法覆盖的一种应用
18. Object- Number - Integer
19. 向上和向下转型
    a. 向上转型可以直接转，向下转型需要在前面加括号（int）long，并且会有转型的风险，可能会报错，会失败
20. 继承到的方法可能是父类的父类的方法
21. 方法重载和方法重写的区别：
    a. 方法重载：在一个类里写多个方法
    b. 方法重写：在子类重写父类的方法
22. 策略模式：用多态实现来代替switch的一种实现方法（很多模式都是用面向对象的特性来重塑一些语言的特性，达到实现低耦合的代码）
